# torsor-helper v0.2 "Intelligence Release" Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans (inline, this session) to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax. TDD throughout: failing test ‚Üí minimal impl ‚Üí green ‚Üí commit. Full suite (`uv run pytest -q`) green between tasks.

**Goal:** Ship 12 verified, dependency-free, offline-testable improvements that make torsor's recall sharper, its map graph-aware, its guard CI-ready, and its coach a prioritizer.

**Architecture:** Extends existing modules (`search`, `indexer`, `cartographer`, `db`, `guard`, `coach`, `operations`) plus three small new modules (`snippets.py`, `export.py`, `baseline.py`, `coach/hotspots.py`). All new persistent state is either rebuildable (`.index/` DB) or committed frontmatter/JSON. No new required deps.

**Tech Stack:** Python 3.11+, stdlib `ast`/`hashlib`/`subprocess`/`math`, NumPy (already a dep), Pydantic v2, Typer, FastMCP, pytest.
